You, Too, Can Run an Automated Test

A Technical Overview of Sauce Labs for Non-Technical Users


Setting Up a Dev Environment

  1. Sign up for a Github account
  2. Open the Gitpod
  3. Sign in to Github & Authorize Gitpod
  4. In the File Drop down, select "Auto-Save"
  5. Open sauce_example.rb file

Starting and Stopping a Sauce Session

  1. Sign up for a Trial Sauce Labs Account
  2. Locate your Sauce Labs Credentials
  3. Paste the credentials where it says "yourusername" and "00000000-0000-0000-0000-000000000000"
  4. Use the Platform Configurator to create your desired combination
    • W3C
    • PC
    • Windows 10
    • Chrome 78
    • Advanced Configuration --> 1024x768
  5. Click the "Ruby" tab at the bottom of the page
  6. Copy/Paste the "capabilities" code over the capabilities line
  7. In the terminal at the bottom, type: ruby sauce_example.rb and hit enter
  8. Look at the job on the Dashboard

Adding Sauce Labs Options

  1. Look at the Test Configuration Options
  2. Scroll down to the very bottom
  3. In the far right column of the last row, copy extendedDebugging information
  4. In the sauce_example.rb file, after the 1024x768, hit "enter" and paste the extendedDebugging line
  5. Uncomment the test and build information
  6. Copy those 3 lines; Click right after "true" after extendedDebugging line and paste
  7. Uncomment the code for finding elements and submitting results
  8. In the terminal, run: ruby sauce_example.rb
  9. Look at the job on the Dashboard
